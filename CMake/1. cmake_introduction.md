#  c++ 为什么会有头文件？
- 头文件头文件完成函数或者对象定义的快速复制粘贴，减少重复，提升工作效率。真正编译链接时用到的只是cpp文件，头文件只是预处理阶段被展开到cpp中，不参与编译。
# c++ 程序的编译流程
编译分四个阶段：预处理、编译、汇编、链接。
- 预处理 ：编译处理宏定义等宏定义（例：#define，#ifdef...#endif等）-- 生成后缀为 “.i” 的文件 。
- 编译 ：将预处理后的文件转换成汇编语言 – 生成后缀为 “.s” 的文件 。
- 汇编 ：将汇编文件翻译为二进制目标文件 – 生成后缀为 “.o” 的文件 。
- 链接 ：多个目标文件（二进制）结合库函数等综合成的能直接独立执行的执行文件 – 生成后缀为 “.out”的文件。
    - 链接分为：静态链接和动态链接。
    - 静态链接
        - 在链接阶段，把所有依赖的静态库文件（.a）内容直接装填到可执行文件中。
        - 在程序执行时，这些代码都会被装入该进程的虚拟地址空间中。
    - 动态链接
        - 在链接阶段，只在可执行程序中记录与动态库（.so）中共享对象的映射信息。
        - 在程序执行时，动态链接库的全部内容被装入进程的虚拟地址空间中。
        - 将链接的过程推迟到运行时，从而提高可执行文件的启动速度。
# 编译和构建
- 编译（compile）: 把源文件编译成可执行文件或者库文件。
- 构建（build）: 编译顺序，即先编译什么，后编译什么。

# make/Cmake
- make: make 本身没有编译和链接功能，是一个批处理工具，通过调用 makefile 文件中的命令来完成编译和链接等操作。makefile 文件定义了编译和链接的规则（顺序等）。
- Cmake: cmake 可以更加简单的生成maekfile文件,然后通过make命令来编译代码。Cmake 工具通过 CMakelist.txt 文件来描述源代码的编译规则，然后 Cmake 会根据这些规则生成 makefile 文件。
- 见下图关系

    CMakelist.txt --------------->CMake
       /|\                          |
        |                           |
        |                           |
        |                           |(CMake 工具根据CMakelist.txt 文件生成 makefile 文件)
        |                           |
        |                           |
        |                          \|/                       （Make 工具根据 makefile 文件调用 gcc 编译器生成可执行文件）
    源文件.cpp -------------------->MakeFile --------------------> Make -----------------> GCC ------------------------>可执行文件
