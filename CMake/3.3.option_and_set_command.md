# option()
## 语法
- option(<variable> "help_string" [initial_value])
    - <variable> 变量名
    - "help_string" 帮助信息
    - [initial_value] 初始值,ON/OFF/TURE/FALSE,默认为OFF
## 示例
```
    # CMakeLists.txt
    cmake_minimum_required(VERSION 3.10)
    project(MyProject)

    # 创建一个选项，允许用户选择是否编译测试程序
    option(BUILD_TESTS "Build test programs" OFF)

    # 创建一个选项，允许用户选择是否开启调试模式
    option(DEBUG_MODE "Enable debug mode" OFF)

    # 根据选项值来设置编译选项
    if(DEBUG_MODE)
        add_compile_definitions(DEBUG_MODE)
        set(CMAKE_BUILD_TYPE Debug)
    else()
        set(CMAKE_BUILD_TYPE Release)
    endif()

    # 添加主库
    add_library(mylib src/main.cpp)

    # 添加可执行文件
    add_executable(myapp src/main_app.cpp)
    target_link_libraries(myapp PRIVATE mylib)

    # 根据选项决定是否编译测试程序
    if(BUILD_TESTS)
        enable_testing()
        add_executable(mytest test/test.cpp)
        target_link_libraries(mytest PRIVATE mylib)
        add_test(NAME MyTest COMMAND mytest)
    endif()
```

# set()
## 语法
- set(<variable> <value> [[CACHE <type> <docstring>] [FORCE]])
    - <variable> 变量名
    - <value> 值
    - CACHE <type> <docstring> 将该变量的值存储在CMake缓存中，以便后续的CMake运行可以读取它。
        - <type> 可以是 FILEPATH, PATH, STRING, BOOL, INTERNAL 等类型之一。
        - <docstring> 是对该选项的描述性文本。
    - FORCE 如果设置了此标志，则会强制覆盖任何先前为该变量设置的值。
## 示例
```
    # CMakeLists.txt
    cmake_minimum_required(VERSION 3.10)
    project(MyProject)

    # 设置一个普通的变量
    set(MY_VARIABLE "Hello World")
    message("普通变量: ${MY_VARIABLE}")

    # 在CMake缓存中设置一个变量
    set(MY_CACHED_VARIABLE "Hello Cache" CACHE STRING "这是一个缓存变量")
    message("缓存变量: ${MY_CACHED_VARIABLE}")

```
